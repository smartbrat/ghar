Color choice must be driven by what the section IS ABOUT, never for visual variety or rhythm. No rainbow pages.

**Why:** User explicitly rejected "alternate colors for cadence" thinking. Page-level coherence comes from restraint — warm-white is the default voice; color is an earned moment. Only sage (community closer) and sand (community) used so far as section-level backgrounds in design-system.html, and that discipline is right.

**How to apply:**

Three levels of color application, in descending boldness:

1. **Whole-section surface** — rare. Reserved for sections whose identity IS that color. Example: sage for community closer, sand for warmth moments. Most sections should NOT get this treatment.

2. **Imagery-on-color tile** — the Color Cards pattern (theme color + masked photo). Great for content-heavy visual sections like Architecture & Design, where imagery itself is the point. Color becomes the stage for the image.

3. **Card accents / specific elements** — badges, category labels, featured highlights, shape accents, data-viz in intel cards. Color shows up inside an otherwise warm-white section.

**Defaults:**
- Warm-white is the canvas. Every section inherits it unless earned otherwise.
- Not mandatory to use all 16 colors — the palette is a resource, not a checklist.
- Before assigning color to a section, ask: "What does this section MEAN, and does that meaning tie to this tone?"
- If the answer is "it just needs variety" — keep it warm-white.
