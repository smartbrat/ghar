Always present the approach and get explicit approval before generating code, especially for:
- Removing content or sections (e.g. Brand Connect category pills — should have asked first)
- Design direction changes (dark → white, layout restructuring)
- Copy/text changes

**Why:** Multiple times code was generated that had to be reverted because the approach wasn't discussed. User values dialogue before action — "Don't build first, let's discuss this."

**How to apply:**
- For design changes: describe what you'd do and why, ask "shall I proceed?" or "what do you think?"
- For removing elements: always ask. Even if something looks like clutter, it might be intentional.
- For copy changes: propose the new text and get approval before editing
- When user shares a screenshot with feedback, understand their intent before jumping to code
- "Share your thoughts and opinion, don't generate anything" = discussion mode, no code
