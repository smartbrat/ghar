**Build sections strictly ONE at a time.** Never propose a plan that builds 2+ sections in one pass, even if the user asks for several. Confirm scope, build ONE, screenshot, get sign-off, then ask before the next.

**Why:** On 2026-05-06 user said "I feel we will have to go through each section at a time as the execution is extremely poor and the usage of shapes looks forced an unnecessary unwanted." All 4 sections built (Editorial, GharTalks, Voices, GharEvents) were thrown out because batching meant compounding mistakes (wrong italic Gazpacho, forced shape glyphs, oversized layouts) before the user could course-correct.

**How to apply:**
- Even if the user lists 4 sections to "quickly finish", treat that as scope, not as instruction to batch. Reply: "Got the list. I'll start with #X — show layout sketch, build, screenshot, you sign off, then I move to next."
- Build → Playwright screenshot at desktop AND mobile → wait for user verdict → only then start the next section.
- If a user says "do all of them" again, confirm with: "I can — but my last batch of 4 had to be wiped. Faster end-to-end if I show you each one and we lock it. OK?"
- This stacks with `feedback_discuss_first.md` (discuss before building) and `feedback_one_at_a_time.md` (don't batch graphic redesigns) — same pattern at section scale.
